# Установка FCS AutoReport

## Для пользователей — один установщик, всё ставится само

Раздавайте **один файл**: **`build\bin\FCS_AutoReport_Setup.exe`**

Пользователь:
1. Запускает **FCS_AutoReport_Setup.exe**
2. Нажимает «Далее» → выбирает папку (или по умолчанию) → «Установить»
3. Всё. Приложение и при необходимости WebView2 ставятся автоматически, ярлыки создаются. Ничего дополнительно ставить, искать или править не нужно.

---

## Автоматическая настройка среды сборки (Go, Wails, NSIS + PATH)

Чтобы **ничего не ставить вручную** и сразу иметь всё для сборки установщика:

1. Откройте папку проекта.
2. Запустите **`setup-build-env.bat`** (двойной клик или из терминала).
3. Скрипт через **winget** установит **Go** и **NSIS**, через **go install** — **Wails**, и **пропишет в PATH** папку `%USERPROFILE%\go\bin` (и при необходимости пути к Go/NSIS).
4. В конце **закройте и снова откройте терминал** (или Cursor), чтобы подхватился обновлённый PATH.
5. Дальше запускайте **`build-installer.bat`** — соберётся установщик.

**Требуется:** Windows 10/11 с **winget** (App Installer обычно уже есть). Права администратора не обязательны (установка в профиль пользователя и запись в пользовательский PATH).

---

## Сборка установщика (если среда уже настроена)

Из корня проекта выполните:

```bash
build-installer.bat
```

или вручную:

```bash
wails build -webview2 embed -nsis
```

В папке `build\bin\` появится **FCS_AutoReport_Setup.exe** — это и есть полноценный установщик (как у обычных программ в Windows). Раздавайте только его.
