# Аудит FCS AutoReport

**Дата:** 12.03.2026

## Архитектура

- **Backend:** Go, Wails v2, SQLite (modernc.org/sqlite), excelize для Excel.
- **Данные:** `internal/db` — БД; `internal/store` — in-memory кэш; `internal/domain` — модели; `internal/app` — сервис, агрегатор, экспорт, Wails-мост.
- **Поток отчёта:** Сырой Excel → `ProcessRawReport` (агрегация, подстановка водителя по городу, имя клиента из колонки «שם לועזי») → `ExportToExcel` (шаблон Минздрава).

## Замечания и рекомендации

### Критичные / исправлено в рамках доработок

1. **Frontend: удаление водителя/товара без проверки бэкенда**  
   При `getBackend()` === null вызов `getBackend().DeleteDriver(agent)` приводил к исключению. Добавлена проверка перед вызовом.

2. **Нет индикации загрузки при генерации отчёта**  
   Пользователь мог нажать «Сгенерировать» несколько раз. Добавлен disabled + текст «Генерация…» на время выполнения.

### Средний приоритет

3. **Справочник клиентов (clients)**  
   В БД и Store есть `clients`, но при формировании отчёта имя клиента берётся только из сырого файла (колонка «שם לועזי»). Справочник клиентов не используется в ETL. Оставить как есть для возможного будущего использования или документировать.

4. **Жёстко заданные индексы колонок в агрегаторе**  
   Номер накладной (4), дата (5), ח"פ (9), адрес (12), артикул (15) и т.д. заданы константами. При смене формата сырого отчёта потребуется правка кода. Рекомендация: при появлении второго формата вынести маппинг в конфиг или определять по заголовкам (как для имени клиента).

5. **Настройки: семантика полей**  
   В `settings` поле `raw_reports_path` фактически хранит путь к **файлу** сырого отчёта, а не к папке. На работу не влияет, при расширении логики «последние файлы» стоит учитывать.

### Низкий приоритет

6. **Логирование в прод**  
   В экспорте и агрегаторе много `slog.Info` (в т.ч. по каждой накладной). Для продакшена можно понизить уровень до Debug или вынести за флаг.

7. **GetDriverForCity**  
   При нескольких водителях на один город всегда возвращается первый из списка. Случайный выбор или приоритеты при необходимости можно добавить позже.

8. **Миграции БД**  
   Миграция `ALTER TABLE drivers ADD COLUMN city_codes` выполняется каждый запуск; ошибка игнорируется (уже есть колонка). Подход допустим для одной миграции; при росте числа миграций лучше версионирование схемы.

## Консистентность

- Имена в domain (PascalCase) и json-теги (snake_case/camelCase) согласованы с фронтом.
- Сохранение настроек: `GenerateReport` вызывает `SaveSettings(rawFilePath, outputDir, templatePath)` — порядок соответствует ожиданиям UI (сырой файл, папка вывода, шаблон).
- Определение колонки клиента в агрегаторе: приоритет «שם לועזי» перед «כתובת» и до «קוד פריט»; запасной вариант — колонка 11, если заголовок подходит. Исключает подстановку названия товара.
- В экспорте колонка «לקוח» ищется по заголовку, с исключением «סוג לקוח».

## Безопасность и надёжность

- Пути к файлам приходят из диалогов ОС (Wails) или из полей ввода — явной санитации путей нет; приложение рассчитано на локальное использование доверенным пользователем.
- SQL: везде параметризованные запросы, инъекции исключены.
- Ошибки от БД и excelize пробрасываются наверх с обёрткой `fmt.Errorf(..., err)`.

## UI (после доработок)

- Кнопка «Сгенерировать отчёт» блокируется на время генерации.
- В журнале есть кнопка «Очистить».
- Модальные окна закрываются по Escape.
- После успешной генерации выводится краткий статус и путь к файлу; при необходимости можно добавить кнопку «Открыть папку».
